The hairline grid

rule 01 · the core trick

Lines are gaps, not borders

The grid container's background is the border color. Cells sit 1px apart and paint their own opaque background, so the container shows through the gaps as perfectly uniform hairlines.

display:grid · gap:1px · background:var(--border)

Why not border on cards?

Per-card borders double up where cards touch, drift at corners, and break at responsive reflow. The gap technique can never produce a doubled or missing line — the geometry does the work, at any column count.

cells: background:var(--background) 
/* the whole trick */
.hairline-grid {
  display: grid;
  gap: 1px;                    /* the "border width"        */
  background: var(--border);    /* the "border color"        */
}
.hairline-grid > * {
  background: var(--background); /* cells mask the container  */
}

Color ladder

rule 02 · 6 greys + 1 accent
backgroundlab(3%) ≈ #0b0b0bpage + cards
surface-raisedlab(7%) ≈ #151515code, chips
surface-hoverlab(9%) ≈ #191919hover fill
borderlab(13%) ≈ #222222all hairlines
fg-mutedlab(73%) ≈ #b3b3b3body, icons
fglab(88%) ≈ #ddddddtitles — not #fff

Narrow value band

All surfaces live between 3% and 13% lightness — depth is whisper-quiet. Text tops out at 88%, secondary text at 73%. Softened whites on near-black is why it reads calm instead of harsh.

One accent, used sparingly

The only chroma on the page: a warm amber, lab(82.6% 15.9 66.4) ≈ #ffc04b. It appears in the headline, the # marks, and focus rings — nowhere else. When mimicking, swap the hue first; it is the most identifiable part of the source.

--accent: lab(82.6% 15.9 66.4)

Typography

rule 03 · two faces, strict roles
display · 60px · w400 · −1.5px Large but light.
section h2 · 24px · w600 · −0.6px Tracking tightens as size grows
card h3 · 20px · w600 · −0.4px Card titles stay compact
body · 14px · fg-muted · lh 1.62 Reading copy is always the sans face, one size, muted grey, relaxed leading. It never carries the mono voice and never brightens to full foreground.
meta · mono · 11–12px GEIST MONO IS RESERVED FOR CHROME: EYEBROWS · COUNTS · VALUES · LABELS

Card anatomy & interaction

rules 04–06 · hover these

Whole card is the link

An anchor styled as a block: 28px padding, title row with a muted ↗ that brightens on hover. No buttons, no "read more" — the affordance is the arrow plus the background lift.

padding:28px · gap:12px

Clamp to three lines

Descriptions are cut at exactly three lines with line-clamp, which is the quiet reason every grid row stays visually even no matter how long the underlying content runs — like this sentence, which is deliberately too long to fit and is being truncated right about now, proving the point.

-webkit-line-clamp:3

One-property hover

The entire hover state is a background shift from 3% to 9% lightness over 150ms. No shadow, no scale, no border change, no translate. Restraint is the effect.

transition:background-color 150ms

Rules of restraint

mimic, don't copy
  • →border-radius: 0, box-shadow: none, everywhere. Flatness is a decision, not an omission. Depth comes only from the hover fill.
  • →Section headers are framed rows, title left and mono count right, sharing the page's hairlines — the whole page reads as one continuous ruled sheet.
  • →Responsive by column count only: 1 column → 2 (→ 3, 6 for swatches). The gap technique survives every reflow untouched.
  • →To make it yours: change the accent hue, swap the type pairing while keeping the sans/mono role split (Inter + JetBrains Mono, etc.), and vary the rhythm with asymmetric spans — a col-span-2 feature cell gives a truer bento while the hairline system holds.
  • →Carry the principles, not the pixels: gap-px hairlines, narrow lightness band, off-white text, three-line clamps, one-property hovers.
